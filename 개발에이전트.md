## 1. 신규 추가: AI 자동 해석 기능

### 1-1. 목적
통계 비전공자가 각 Step의 결과(ATE, CATE, 신뢰구간, 반사실 등)를 이해하도록,
LLM이 결과를 **2~3문장의 쉬운 한국어**로 자동 설명한다.

### 1-2. 설계 결정사항
| 항목 | 결정 |
|------|------|
| 위치 | 각 Step 결과 **바로 아래 자동 표시** |
| 범위 | 모든 Step (0~7) |
| 길이 | 2~3문장 |
| 톤 | 완전 비전공자 수준 |
| 지식원 | PPT '인과추론 적용분야' 9장의 번역 사전을 SYSTEM 프롬프트에 내장 |

### 1-3. 파일 구조 (신규)
- **`llm.py`** — LLM 추상화 계층
  - 로컬: Ollama HTTP API (`http://localhost:11434`, 모델 `gemma3:4b`)
  - 클라우드: Gemini API (`st.secrets["GEMINI_API_KEY"]`, 모델 gemini-2.5-flash-lite → flash)
  - **자동 전환**: Ollama 응답 없으면 Gemini로 fallback (`detect_mode()`)
  - Gemini 일일 한도 20회 (`data/usage.json` 추적), 환경변수 `DAILY_LIMIT`로 조정
  - 모델 교체는 환경변수 `OLLAMA_MODEL` 한 줄로 가능
- **`explain.py`** — Step별 해석 생성 + Streamlit 렌더링
  - `VAR_LABELS`: 변수 약어→한글 사전 (단일 진실)
  - `vlabel()/vlabels()`: 약어를 한글 라벨로 변환
  - `SYSTEM`: 변수 사전 + 환각 금지 + 톤 규칙 내장 프롬프트
  - `show(cache_key, prompt)`: 캐시(session_state) 후 박스 렌더링. 동일 내용 재호출 안 함.

### 1-4. app.py 통합 지점
- `import explain` 추가
- 사이드바: "AI 해석 자동 표시" 토글 + 엔진/사용량 표시
- Step 0~7 각 결과 끝에 `explain.show(...)` 호출 (총 8개 지점, Step 4는 다중T/단일/매개 분기별)

### 1-5. requirements.txt 추가
```
requests>=2.31.0
google-genai>=0.3.0
```

## 2. AI 해석 정확도 보완 (3차에 걸쳐)

소형 모델(gemma3:4b)이 변수 의미를 모르고 환각·방향역전을 일으키는 문제를 단계적으로 해결.

### 2-1. 1차 — 기본 구현
- 각 Step에 해석 삽입. 단, 변수 사전 미내장 상태.

### 2-2. 2차 — 환각·방향오류 제거
| 문제 | 원인 | 해결 |
|------|------|------|
| STEP2 "가격", STEP4 "앱 사용 시간" 환각 | 변수 의미 모름 + PPT 일반예시(가격/프로모션)가 환각 유도 | 변수 사전 SYSTEM 내장 + "사전에 없는 소재 지어내지 마라" + 영문약어 출력 금지 |
| W를 교란변수로 이해 못함 | 용어 불일치 | "교란변수=배경 요인" 통일 |
| STEP6 방향 거꾸로 | 4B 모델 방향 역전 | 방향(올라간다/내려간다)을 Python에서 미리 확정해 완성문장 전달 |

### 2-3. 3차 — 서브그룹·유의성·억지해석 보완
| 문제 | 해결 |
|------|------|
| STEP6 서브그룹 필터별 차이 무시 | 서브그룹 설명을 cache_key·프롬프트에 포함. 범위가 낮은편/높은편/중간인지 median 비교로 라벨링 |
| STEP4 불확실 효과에서 원인↔결과 역전 | 유의하지 않으면 "뚜렷한 효과 없음, 방향 단정 어려움"으로 정직하게 프레이밍 |
| STEP5 "시스템 품질" 환각 | T_list·Y 라벨을 프롬프트에 전달 + "방향/크기 언급 금지, 새 변수 금지" |
| STEP6 미미효과(+0.11) 억지 의미부여 | \|delta\|<0.15면 "거의 차이 없음"을 앞세움 + "이유 추측 금지" |
| 다른 변수 끌어와 추측 | SYSTEM에 "주어진 변수만 언급, 다른 변수 추측 금지" 규칙 추가 |

### 2-3-B. 4차 — 변수 반대의미 의역 차단
| 문제 | 해결 |
|------|------|
| STEP6에서 '인지된 위험 증가'를 '안전하다고 느끼는 정도 증가'로 뒤집어 의역 | SYSTEM에 "변수를 반대 의미 단어로 바꾸지 마라. '인지된 위험↑'은 '더 위험'이지 '안전해짐'이 아니다" 규칙 + 변수사전에 "값↑=더 위험=덜 안전" 명시 |

### 2-4. 핵심 기법 요약
1. **변수 사전을 SYSTEM에 내장** — 환각의 근본 차단
2. **방향·부호 해석은 Python에서 확정** — 모델은 다듬기만 (역전 방지)
3. **불확실/미미한 효과는 정직하게 "효과 없음"** — 억지해석 차단
4. **서브그룹·T·Y를 프롬프트와 캐시키에 명시** — 맥락 인식
